# Contributing to the User Guide

Thank you for contributing to the Scions of Silverymoon User Guide!

## Content Guidelines

### File Structure

- Place new guide pages in the `_guides/` directory
- Use `.md` (Markdown) format for all content
- Name files using lowercase with hyphens (e.g., `getting-started.md`)

### Markdown Format

Each guide page should include front matter at the top:

```yaml
---
title: Page Title
description: Brief description of the page
order: 1
---
```

### Writing Style

- Write in clear, concise language
- Use active voice
- Include examples where helpful
- Keep the target audience (server members and players) in mind
- Maintain a friendly and welcoming tone

### Making Changes

1. Create a new branch for your changes
2. Make your edits to the appropriate files
3. Test locally if possible (see README.md)
4. Submit a pull request with a clear description of your changes
5. Wait for review from staff members

### Review Process

- All content changes should be reviewed by at least one staff member
- Ensure changes align with server policies
- Check for spelling, grammar, and formatting errors

## Questions?

If you have questions about contributing, please reach out to the server staff.
